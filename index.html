FAQ

Frequently asked questions

Do I need a new AI subscription?
  • No new subscription required.
  • Bring the subscription you already use.
  • DroidMaster supports Codex for OpenAI and Claude Code headless.
Can I use it with Claude Code?
  • Yes. DroidMaster wraps claude -p.
  • That is the headless version of Claude Code.
  • It uses approved official Claude tooling, not API workarounds.
How does OpenAI subscription support work?
  • OpenAI subscription support uses Codex.
  • DroidMaster adds faster workspace navigation around it.
  • Files, browser context, and voice prompting stay in one surface.
Can I use Codex and Claude at the same time?
  • Yes. DroidMaster supports multi-account auth.
  • Create a conversation with any supported provider.
  • Keep separate provider identities available in the same app.
What can DroidMaster access locally?
  • Only what you expose.
  • Choose the projects, folders, and credentials available to the container.
  • Grant runtime access only where you feel safe.
Can it generate images?
  • Yes. Image generation tools are included through OpenRouter.
  • Generated media lives next to code, files, chat history, and browser work.
  • The image output stays inside the same operator workspace.
Can I use it from mobile?
  • Yes. Deploy the app to cloud.
  • Connect the mobile app.
  • Keep the remote surface synced with your local workspace.
What upcoming features are planned?
  • Local voice transcription support.
  • More coding agents in the provider roster.
  • Choice between a local LLM and a cloud provider per workspace.
Is DroidMaster only for solo power users?
  • No. The open-source path is built for operators.
  • The enterprise path adds collaborative content pipelines.
  • Reviews, assets, and handoffs share the same operating surface.
Why include an in-app browser?
  • The browser keeps inspection and change requests together.
  • Review the live product state.
  • Prompt the agent without switching contexts.
Can DroidMaster run locally or in the cloud?
  • Yes. Run locally for scoped machine access.
  • Deploy to your own infrastructure for remote access.
  • Use cloud mode when you need mobile continuity.
